# Pose Deck — iOS

Native SwiftUI app (iOS 17+, Swift 5.9+) for the M2 prep MVP. The app target is
a thin SwiftUI shell; all testable logic lives in the **PoseDeckCore** Swift
package (`PoseDeckCore/`).

## Layout

```
ios/
├── PoseDeck/              # App target (generated Xcode project)
│   ├── project.yml        # xcodegen spec — the source of truth for the project
│   ├── Config/
│   │   ├── Config.xcconfig # API_BASE_URL and other build settings
│   │   └── Info.plist      # generated by xcodegen from project.yml
│   ├── Sources/           # App Swift sources (PoseDeckApp, ContentView, AppConfig)
│   └── Tests/             # App unit-test target stub
└── PoseDeckCore/          # Logic core Swift package (models, APIClient, outbox)
```

## Prerequisites

- Xcode 16.3+
- [xcodegen](https://github.com/yonaskolb/XcodeGen) (`brew install xcodegen`)

## Generating & opening the project

`PoseDeck.xcodeproj` is **generated** from `PoseDeck/project.yml` and is **not**
committed to git (see `ios/.gitignore`). After cloning, or any time you change
`project.yml` / add or remove source files, regenerate it:

```bash
cd ios/PoseDeck
xcodegen generate
open PoseDeck.xcodeproj
```

Then build/run the `PoseDeck` scheme from Xcode.

## Logic core tests

```bash
cd ios/PoseDeckCore
swift test
```

## Dev API base URL

The backend (PocketBase) base URL is configured in
`PoseDeck/Config/Config.xcconfig` via `API_BASE_URL` (default
`http://localhost:8090`). It is surfaced into `Info.plist` as `APIBaseURL` and
read at runtime by `AppConfig.apiBaseURL`.

To target a LAN backend (e.g. a TrueNAS host), edit `Config.xcconfig`:

```
API_BASE_URL = http:$(SLASH)$(SLASH)192.168.1.50:8090
```

> The `$(SLASH)` indirection is an xcconfig idiom: `//` starts a comment in
> xcconfig, so the scheme's slashes are injected via a helper variable to avoid
> the value being truncated to `http:`.

App Transport Security allows local cleartext HTTP (`NSAllowsLocalNetworking`)
for the dev backend — review before any production build.

Dev seed users (backend): `owner@posedeck.test` / `changeme123`,
`guest@posedeck.test`.
